# 业务逻辑文档：众筹、医疗与志愿者

本文档详细说明了系统中关于 **动物模型**、**医疗众筹流程** 以及 **志愿者任务** 的核心业务逻辑。

---

## 1. 动物模型 (Animal Model)

`Animal` 是系统的核心实体，它不仅记录了动物的基本信息，还通过状态字段驱动了众筹和医疗流程。

### 核心字段

| 分类 | 字段名 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| **基本信息** | `Name` | string | 动物昵称 |
| | `Species` | string | 物种 (Dog / Cat) |
| | `Breed` | Breed | 品种关联 |
| | `Gender` | string | 性别 (Male / Female / Unknown) |
| | `Size` | string | 体型 (Small / Medium / Large) |
| | `EstAge` | string | 估计年龄 (e.g. "2 years") |
| **状态** | `Status` | string | 总体状态 (Stray: 流浪中 / Adopted: 已领养) |
| | `LocationName` | string | 当前位置描述 |
| **医疗健康** | `IsVaccinated` | bool | 是否已接种疫苗 |
| | `IsNeutered` | bool | 是否已绝育 |
| **众筹相关** | `CurrentGoalType` | string | 当前众筹目标 (None / Vaccine / Neuter) |
| | `GoalAmount` | decimal | 目标金额 |
| | `RaisedAmount` | decimal | 已筹集金额 |
| | `GoalStatus` | string | 众筹状态 (**关键状态机**) |

### 众筹状态 (`GoalStatus`) 说明

1.  **`Fundraising`** (募资中): 正在接受用户捐款。
2.  **`ReadyForVolunteer`** (待认领): 资金已凑齐，等待志愿者认领任务。
3.  **`Completed`** (已完成): 医疗任务结束，健康状态已更新。

---

## 2. 医疗众筹全流程 (Medical & Donation Workflow)

这是一个自动化的闭环流程，从设置目标开始，到医疗任务完成结束。

### 阶段 1: 开启众筹 (管理员/Shelter)
调用 `MedicalService.SetGoalAsync`：
1.  设置 `CurrentGoalType` (如 "Vaccine") 和 `GoalAmount` (如 100.00)。
2.  系统自动将 `RaisedAmount` 重置为 0。
3.  系统将 `GoalStatus` 设为 **`Fundraising`**。

### 阶段 2: 用户捐款 (User)
用户调用 `MedicalService.DonateAsync`：
1.  **余额检查**: 检查用户余额是否充足。
2.  **扣款**: 扣除用户余额，记录负数 `WalletTransaction` (Type: Donation)。
3.  **记录**: 创建 `Donation` 记录。
4.  **进度更新**: 增加动物的 `RaisedAmount`。
5.  **自动触发**:
    - 如果 `RaisedAmount >= GoalAmount`，系统自动将 `GoalStatus` 更新为 **`ReadyForVolunteer`**。
    - 此时众筹结束，任务进入任务池。

---

## 3. 志愿者任务流程 (Volunteer Workflow)

当动物状态变为 `ReadyForVolunteer` 后，志愿者介入。

### 阶段 1: 认领任务 (Claim)
志愿者调用 `MedicalService.ClaimTaskAsync` (或 `VolunteerService.ClaimTaskAsync`)：
1.  **前置检查**: 确认动物 `GoalStatus` 为 `ReadyForVolunteer`。
2.  **任务创建**: 创建 `MedicalTask` 记录。
    - `TaskType`: 对应动物的 `CurrentGoalType`。
    - `Status`: **`InProgress`**。
    - `VolunteerId`: 绑定当前志愿者。

### 阶段 2: 执行与提交 (Complete)
志愿者带动物去医院完成任务后，调用 `VolunteerService.CompleteTaskAsync`：
1.  **上传证明**: 上传现场照片 (`ProofImage`) 和收据 (`ReceiptImage`)。
2.  **填写费用**: 填写实际花费 `ExpenseAmount`。
3.  **提交审核**: 任务状态变为 **`PendingApproval`**。

### 阶段 3: 管理员审批 (Approve)
管理员调用 `MedicalService.ApproveTaskAsync`：
1.  **状态更新**: 任务状态变为 **`Approved`**。
2.  **健康更新**: 根据任务类型自动更新动物数据。
    - 如果是 Vaccine -> 设置 `Animal.IsVaccinated = true`
    - 如果是 Neuter -> 设置 `Animal.IsNeutered = true`
3.  **闭环**: 将动物的 `GoalStatus` 设为 **`Completed`**。

---

## 4. 放弃任务 (Abandon)
如果志愿者认领后无法完成：
- 调用 `VolunteerService.AbandonTaskAsync`。
- 任务 `Precondition`: 必须是 `InProgress` 状态。
- 结果: 解除志愿者绑定，任务状态重置为 `Open`，其他志愿者可重新认领。
